# 우리 시스템 — 3분 설명용

> 이 기술을 처음 듣는 사람에게 소개하는 문서. 전문용어 최소화.

---

## 한 줄 요약

> **창업 아이디어를 6명의 AI 전문가가 3번씩 검토하고, 판사 AI 가 최종 점수를 매기는 법원 같은 시스템.**

그냥 ChatGPT 에게 "이 아이디어 어때?" 물어보는 것보다 **훨씬 믿을 만한 평가** 를 만들기 위한 구조.

---

## 왜 이게 필요한가

ChatGPT 한테 "내 스타트업 아이디어 어때?" 물어보면 생기는 문제:

1. **칭찬만 함** — AI 가 정중해서 약점을 얼버무림
2. **같은 질문에 매번 다른 답** — 점수 달라짐, 믿기 어려움
3. **근거 없이 말함** — "시장이 크다" 라고만 하고 출처 없음
4. **한 번 답하면 끝** — 실수를 스스로 안 고침
5. **단순 비판은 무조건 깎음** — 보수적으로 과소평가된 좋은 아이디어를 놓침

우리는 **이 5개를 각각 구조적으로 해결**했다.

---

## 어떻게 작동하나 (4단계)

```
[사용자 아이디어 입력 — 6 axis 답변]
        ↓
┌──────────────────────────────────────┐
│ 1단계: 문지기 (Gate)                  │
│  "설명이 너무 부실하지 않은가?"        │
│  통과 못 하면 여기서 막음              │
└──────────────────────────────────────┘
        ↓
┌──────────────────────────────────────┐
│ 2단계: 6명의 전문가가 각자 3번 평가     │
│                                      │
│  ① 시장 전문가     ② 문제 강도 전문가  │
│  ③ 타이밍 전문가   ④ 제품 (10x) 전문가 │
│  ⑤ 해자 전문가     ⑥ 비즈니스 모델 전문가│
│                                      │
│  각자가 3단계로 판단:                  │
│   → 초안 작성 (낙관적)                 │
│   → 균형 보정 (과장이면 ↓, 과소평가면 ↑)│
│   → 최종 판결 (근거 검증)              │
└──────────────────────────────────────┘
        ↓
┌──────────────────────────────────────┐
│ 3단계: 서사 조립                      │
│  6명의 평가를 읽기 좋은 리포트로 정리    │
│  ※ 점수는 절대 안 바꿈 — agent 원본 고정│
└──────────────────────────────────────┘
        ↓
┌──────────────────────────────────────┐
│ 4단계: 최종 점수 공식 적용              │
│  6축 점수에 가중치를 곱해 합산          │
│  (각 축의 중요도가 다름)               │
└──────────────────────────────────────┘
        ↓
   최종 리포트 + 점수
```

---

## 각 단계를 일상 비유로

| 단계 | 비유 |
|---|---|
| 1단계 문지기 | 영화관 입장권 검사 — 입장권 없으면 못 들어감 |
| 2단계 6 전문가 × 3단계 | **대학원 논문 심사** — 6명 심사위원이 읽고 (초안), 균형 잡고 (보정), 최종 판정 (확정) |
| 3단계 서사 조립 | **기자의 기사 쓰기** — 취재 자료 (6 전문가 의견) 를 독자가 읽기 좋게 조립 |
| 4단계 점수 공식 | **성적 계산기** — 과목별 점수에 학점 가중치를 곱해 평점 산출 |

---

## 다른 AI 서비스랑 뭐가 다른가

| | ChatGPT 에게 묻기 | **우리 시스템** |
|---|---|---|
| AI 호출 횟수 | 1번 | **19번** (분야별 + 보정) |
| 점수 재현성 | 매번 다름 | 같은 입력 → 같은 점수 |
| 근거 제시 | 없음 | **웹 검색 URL** 필수 |
| 약점 발견 | 놓침 | **"보정 전담 AI"** 가 따로 있음 |
| 과소평가 잡기 | 없음 | **양방향 보정** (점수가 올라갈 수도) |
| 최종 점수 산출 | AI 가 생각으로 계산 | **수학 공식** 으로 자동 계산 |
| 분석 시간 | ~5초 | ~30-40초 |
| 분석 비용 | $0.001 | ~$0.05 (50배) |

비용·시간 더 들지만 **판단 품질은 전혀 다른 수준**.

## 재미있는 부분들

### "보정 전담 AI" 가 따로 있다 (Calibrator)
- 6명의 전문가 각자에게 **그림자 역할**이 붙어있어서 초안을 검토한다
- **단순 비판이 아님** — 낙관 과장이면 깎고, 보수적으로 과소평가된 강점이면 올린다
- 보정자는 점수 결정권 없음. 오직 보정 제안만 한다
- 판사 AI 가 원본 의견 + 보정을 모두 보고 최종 결정 (`draft × 0.4 + calibrator × 0.6`)

### AI 가 지어낸 가짜 URL 은 자동 제거
- AI 는 "그럴듯하지만 존재하지 않는" URL 을 만들어내는 버릇이 있음
- 우리는 실제로 웹에서 검색한 URL 만 화이트리스트로 받음
- AI 가 가짜 URL 인용하면 **코드가 자동으로 걸러냄** → 할루시네이션 0

### 점수는 코드가, 서사는 LLM 이
- LLM 은 산수를 못 한다 — 가중평균을 부탁하면 매번 다른 값이 나옴
- 우리는 6축 점수를 **코드의 가중 산술 평균** 으로 계산
- LLM 은 점수를 만지지 못함 — 오직 자기 axis 분석만 한다
- → 같은 입력 → 같은 점수 (재현성)

### 실시간으로 평가 과정이 보인다
- 6명의 전문가가 평가하는 동안 사용자는 기다림만 보지 않음
- 각 전문가가 평가를 마치면 즉시 화면에 점수가 채워짐
- "시장 전문가: 78점 ✓" → "문제 전문가: 62점 ✓" 식으로 순차 등장

---

## 언제 이런 시스템이 필요한가

- 수십 개의 창업 아이디어를 비교해야 하는 VC / 액셀러레이터
- 사내 신사업 제안을 평가하는 대기업 이노베이션팀
- 자기 아이디어가 투자할 가치 있는지 객관적으로 알고 싶은 예비 창업자
- 시장 진입 전 "이게 정말 말이 되나?" 체크가 필요한 기획자

"감 / 경험 / 직관" 에 의존하던 평가를 **구조화 + 재현 가능 + 증거 기반** 으로 바꾸는 시도.

---

## 3분 요약

1. **6명의 전문가 AI** 가 스타트업 아이디어를 각자 담당 분야별로 평가 (시장 / 문제 / 타이밍 / 제품 / 해자 / 비즈니스 모델)
2. 각 전문가는 **초안 → 양방향 보정 → 최종 판결** 3단계로 신중하게 판단 (총 18번의 AI 호출)
3. **웹 검색** 으로 실제 근거 URL 확보, 지어낸 가짜는 코드가 자동 제거
4. 모든 판단이 끝나면 **수학 공식 (가중 산술 평균)** 으로 최종 viability 점수 산출 (5-95)
5. **단순 비판이 아니라 양방향 보정** — 과대평가도 과소평가도 잡음

→ 그냥 AI 에게 묻는 것보다 **훨씬 믿을 만한, 재현 가능한, 근거 있는 평가**.
