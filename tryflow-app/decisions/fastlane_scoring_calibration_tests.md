# Fastlane 채점 보정 테스트 데이터셋

3-Pass (Draft → Skeptic → Judge) 파이프라인 + rubric 자동 생성 결과를 검증하기 위한 양 극단 테스트 세트. 같은 대회에 두 출품작을 넣어 점수 격차를 확인한다.

작성 의도: 한국 평가 문화의 보수성 (평균 50, 80+ 는 드물게) 이 실제로 작동하는지 + 출품작 완전성 미충족 시 anchor ~10/~25 로 강제 하향되는지 검증.

도메인: **공익광고 공모전 — 청년 정신건강** (창의/사회 임팩트 영역, 알고리즘과 완전 다른 결).

---

## 1️⃣ 대회 설정 (`/competitions/new`)

| 필드 | 값 |
|---|---|
| **대회명** | `2026 청년 정신건강 인식개선 공익광고 공모전` |
| **주최 기관** | `한국정신건강재단 · 보건복지부` |
| **주제 / 도메인** | `공익광고 캠페인 - 청년 정신건강` |
| **제출 마감** | 오늘 +30 일 |

### 평가 항목 (가중치 합 100%)

| 항목명 | 가중치 | 채점 기준 설명 |
|---|---|---|
| 인사이트의 정확성 | 30 | 청년 정신건강의 실제 현실과 진입 장벽에 대한 깊은 이해 |
| 메시지의 명료성 | 25 | 핵심 메시지가 한 줄로 명확하고 기억에 남게 전달되는지 |
| 크리에이티브 차별성 | 25 | 기존 정신건강 캠페인과 구분되는 새로운 표현·접근 방식 |
| 실행 가능성 | 20 | 채널 전략·예산 구조·실행 디테일이 현실적인지 |

저장 후 rubric 자동 생성 대기 (~30 초). "rubric 준비 완료" 배너 뜨면 출품 제출 가능.

## 2️⃣ 출품작 A — `~10점 예상` (정보 부족 케이스)

### 제목
```
청년 멘탈 캠페인
```

### 팀명 / 출품자
```
학생A
```

### 본문 / 요약
```
청년들 우울증 많다고 함. 도움 받으라고 광고할거임. 인스타로 함. 1357 같은 상담 번호 넣을거.
```

### 예상 채점 흐름
- **출품작 완전성 검증** (rubric 첫 체크포인트) → 타겟 페르소나 X / 인사이트 데이터 X / 핵심 메시지 X / 크리에이티브 컨셉 X / 예산 X / KPI X — 모두 누락
- **Draft** → ~20 점 (한 줄 컨셉만, anchor ~10/~25 와 매칭)
- **Skeptic** → 과장 지적: "캠페인 본문에 평가 가능한 요소가 거의 없음, 인스타 + 상담번호 외 구체성 없음" → suggested ~12
- **Judge** → 가중평균 + anchor 검증 → **15~22 점대**

## 3️⃣ 출품작 B — `~90점 예상` (모든 요건 충족)

### 제목
```
괜찮니, 진짜로? — 직장인 점심시간 3분 인터뷰 리얼리티 광고
```

### 팀명 / 출품자
```
마음챙김 스튜디오
```

### 본문 / 요약
```
[타겟 페르소나]
직장 생활 3-5년차 (27-32세) 사무직 청년. 매일 "안녕하세요?" "괜찮아요" 자동 응답으로 지내지만, 내적 피로와 소진감이 누적된 상태. 정신건강 검진의 필요성은 인식하지만 "약 먹어야 할 정도는 아니야" 라는 자기검열로 진입 장벽이 형성되어 있다.

[핵심 인사이트]
2024년 보건복지부 청년정신건강 실태조사: 20-30대 우울 위험군 18.3%, 그중 의료적 도움을 요청한 비율은 7.4% 미만. 가장 큰 진입 장벽은 "병원에 갈 정도는 아니다" 라는 자기검열 (37%). 즉 캠페인의 핵심 task 는 "병원 가야 함" 을 설득하는 것이 아니라, "내 상태를 잠시 점검해도 된다" 는 허용감을 제공하는 것이다.

[캠페인 컨셉]
"괜찮니, 진짜로?" — 매일 의례적으로 주고받는 "괜찮아요" 에 잠시 멈춤을 주는 한 줄.
점심시간 3분 동안 사무직 직장인 실제 인물이 카메라 앞에서 마주 앉아, 동료 한 명에게 "괜찮니, 진짜로?" 만 물어보는 리얼리티 영상 시리즈. 답변은 편집 없이 길든 짧든 그대로 송출한다.

[크리에이티브 디렉션]
- 30초 / 90초 / 풀버전(3-5분) 3가지 컷 제작
- 흑백 처리 X. 회의실·카페·옥상 등 진짜 점심 공간 그대로
- 자막 최소화 — 침묵과 어색함을 강조
- 매 영상 끝 한 줄 CTA: "내 상태를 점검할 권리, 1357 정신건강위기상담"

[채널 전략]
- Instagram Reels: 30초 컷 — 알고리즘 도달 우선
- YouTube Shorts + 풀버전: 검색 유입
- LinkedIn: 90초 컷 — 직장인 타겟 정밀
- 점심시간 (11:30-13:30) 시간대별 sponsored 노출 집중
- 4주 캠페인: 매주 화/목 신규 1편 (총 8편)

[예산 — 총 1억 2천만 원]
- 콘텐츠 제작: 4,800만 원 (8편 × 600만)
- Reels/Shorts 광고 운영: 3,500만 원 (CPM 12K 기준, 약 290만 도달)
- LinkedIn sponsored: 1,500만 원
- 인터뷰이 사례비 + 진행 PD: 1,400만 원
- 사전/사후 측정 리서치: 800만 원

[KPI 및 측정]
- 도달 (Reach): 누적 250만 (자연+유료)
- 인게이지먼트: 평균 좋아요+댓글+공유율 4.5% 이상 (정신건강 캠페인 벤치마크 2.1%)
- 행동 KPI: 1357 상담 전화·온라인 채널 유입 증감 (캠페인 기간 월 평균 +25% 목표)
- 인식 조사: 캠페인 전후 청년 1,000명 패널 — "내 정신건강 상태 점검의 허용감" 5점 척도 +0.8점 이상

[기존 캠페인 대비 차별점]
정부 정신건강 캠페인은 대부분 "당신은 혼자가 아니에요" 류 위로 메시지 + 의료기관 안내 구조다. 이는 도움이 절실한 사람에겐 닿지만, "괜찮은 척 하는" 다수 직장인에겐 무관한 메시지로 스킵된다. 본 캠페인은 메시지 자체를 한 단계 앞에 둔다 — "병원 가야 함" 이 아닌 "내가 괜찮은지 점검해도 된다" 는 진입의 허용. 의료적 행동 변화 이전의 인지적 변화에 초점을 둠으로써 도달 가능 인구를 18.3% (우울 위험군) → 60%+ (자기검열 그룹) 로 확장한다.
```

### 예상 채점 흐름
- **출품작 완전성 검증** → 타겟 ✅ / 인사이트 ✅ (데이터 인용 포함) / 컨셉 ✅ / 크리에이티브 ✅ / 채널 ✅ / 예산 ✅ / KPI ✅ — 모두 충족
- **Draft** → 82~90 (인사이트의 깊이: 자기검열 메커니즘 / 차별성: 메시지 위치 한 단계 앞으로 / 명료성: 한 줄 컨셉 / 실행: 구체 예산 + 측정)
- **Skeptic** → 명시적 근거 다수, 큰 과장 없음. suggested 약간 보정 (~85) — "벤치마크 수치의 출처가 일부 모호" 정도 지적
- **Judge** → **85~92 점대**

## 4️⃣ 검증 체크리스트

테스트 시 확인할 것:

- [ ] 출품작 A 의 종합 점수가 **30 점 이하**
- [ ] 출품작 B 의 종합 점수가 **80 점 이상**
- [ ] 점수 격차가 **50 점 이상**으로 벌어짐
- [ ] A 의 항목별 σ (Draft/Skeptic/Judge 분산) 가 작음 (세 agent 가 "정보 부족" 으로 합의)
- [ ] B 의 항목별 σ 가 작음 (셋 다 우수성 동의)
- [ ] A 의 axis_reports 에 "출품작 완전성 미충족" / "정보 부족" 류 분석이 명시
- [ ] B 의 axis_reports 가 각 평가 기준의 구체적 충족 근거를 인용
- [ ] B 의 종합 리포트에 **축 간 통찰 (Cross-Axis Insights)** 섹션이 나타남
  - 예시: "인사이트의 깊이 (자기검열 메커니즘 발견) + 메시지의 명료성 (한 줄 컨셉) 이 결합되어 도달 가능 인구를 18% → 60% 로 확장하는 차별성을 만든다 — 두 axis 가 상호 보강하는 패턴."
- [ ] B 의 종합 리포트에 **출품자 보완 방향** 섹션이 구체 가능한 항목 3-5개 포함
- [ ] axis_markdown 의 분량이 700-1200자 (이전 250-450자 대비 증가 확인)
- [ ] 종합 report_md 의 분량이 2000-3500자 (8개 섹션 모두 등장)

## 5️⃣ 보정 시나리오

### 만약 A 가 여전히 50+ 나오면
→ rubric 의 완전성 체크포인트가 효력을 못 내는 거. 두 가지 조치:
1. 대회 detail 페이지 "재생성" 버튼으로 rubric 다시 생성 (`prompts_competition/rubric_generator.md` 의 최신 가이드 적용)
2. 그래도 안 되면 `app/api/competitions/[id]/proposals/[pid]/evaluate/route.ts` 의 Draft system prompt 의 "정보 부족 → 35 이하" 룰을 더 강하게 박기

### 만약 B 가 60 대 나오면
→ 너무 짜졌음. 두 가지 조치:
1. `prompts_competition/rubric_generator.md` 의 Calibration Anchor 가이드 살짝 완화 (~88 anchor 의 "상위 5%" 표현을 "상위 10-15%" 로)
2. Judge 의 "65+ 줄 수 없다" 룰의 조건을 더 명확히

### 점수 격차가 30 점 이내면
→ Draft/Skeptic 의견이 통합 안 됨. `agent_judge.md` 의 가중 공식 (`base = draft × 0.4 + skeptic × 0.6`) 을 조정.

### B 의 종합 리포트가 짧으면
→ `prompts_competition/proposal_synthesizer.md` 의 분량 강제 (2000-3500자) 가 무시됨. response_format 또는 max_tokens 설정 확인.

## 6️⃣ 비고

- `prompts_competition/rubric_generator.md` 가 도메인별 "출품작 완전성 체크포인트" 를 자동 인코딩하므로, 공익광고 외에도 미술·소설·과학·창업 등 어떤 도메인이든 유사 패턴으로 ~10/~90 테스트 가능.
- 같은 출품작을 여러 번 평가해도 점수 드리프트가 ±5 이내여야 함 (3-Pass 결정성 + temperature 단계적 하강 0.6 → 0.3 → 0.2).
- 이 문서는 점수 튜닝 작업 시 reference 로 사용 — 새 보정 후 동일 테스트로 회귀 검증.
- 도메인 변경 테스트 시 본 문서를 복사한 뒤 대회 설정과 출품작 본문만 갈아끼우면 됨.
