### 샘플 A — **고품질 출품작** (예상 종합 70-80)

**Title**:
```
무한루프 다이얼 — 시간 루프를 도구로 사용하는 퍼즐 플랫포머
```

**Team**:
```
스튜디오 노바
```

**Summary**:
```
무한루프 다이얼은 "시간 루프에 갇힌 주인공이 탈출하는 게임" 이 아니라, "주인공이 의도적으로 루프를 만들어 적과 환경을 가두는 퍼즐 플랫포머" 다. 대회 테마 "Stuck in a Loop" 를 정반대 방향 (L4 전복) 으로 해석.

핵심 메커닉: 플레이어는 다이얼을 돌려 자기 행동 5초 구간을 루프로 만들 수 있다. 만들어진 루프는 환경 오브젝트로 작동 — 적을 가두거나, 자기 자신의 과거 행동을 디딤돌로 사용하거나, 스위치를 영원히 누른 상태로 유지한다. 메인 메커닉 1개 + 변주 (체이닝, 중첩 루프) 가 30분간 점진 학습된다.

레벨 디자인: 단계당 5-7개의 짧은 챔버 (Portal 영향). 각 챔버는 새 변주를 1개씩 가르치고 4-6번 변주한 뒤 다음 변주로 조합. 총 24개 챔버 + 4개 보스 퍼즐. 30-45분 분량. 후반 챔버는 루프 3개 동시 운용 요구.

분위기: 검은 배경에 진공관 발광 미감 (Outer Wilds 영향). 시간을 다루는 게임 특유의 멜랑콜리 + 신비 톤. 환경 오브젝트마다 짧은 회상 텍스트 — 사라진 시간 정비공의 일지.

스토리: 단편 vignette (60-90분). 메인 캐릭터 (시간 정비공) 의 부재한 동반자가 환경 오브젝트로 입체적으로 드러남. show don't tell 충실. 결말 2종 (루프 해체 / 영원한 보존), 마지막 챔버의 마지막 선택지로 분기.

팀: 디자이너 1 + 프로그래머 1 + 컴포저 외주 1. 개발 기간 5개월. 1개월차 vertical slice 완성 (3개 챔버), 3개월차 alpha (12개 챔버), 4개월차 beta + 외부 플레이테스트 2회 (각 N=10), 5개월차 polish + 출시. Cut list 명시: 시간 부족 시 챔버 24 → 18 축소, 결말 분기 단일화.

QA 계획: Git + GitHub Actions (PR 마다 자동 빌드), Trello 이슈, Windows/macOS 호환 매트릭스, 60fps@1080p 목표, Sentry 크래시 리포팅. 영/한 localization.

시장: 타겟 = "Portal/Braid 팬, 짧고 깊은 인디 퍼즐 선호". Steam 출시 가격 $12, 데모 Steam Next Fest 참여, 스트리머 20명 outreach 명단. Reference: "Portal 의 챔버 페이싱 + Braid 의 시간 메커닉 + Outer Wilds 의 톤". 리텐션 = 1회차 90분 + ng+ 1회차 (다른 결말).
```

## 4. 평가 결과 검증 기준

각 샘플 평가 완료 후 다음을 확인:

### 4.1 점수 일관성
- [ ] 샘플 A 종합 > 샘플 B 종합 > 샘플 C 종합 (단조 감소)
- [ ] 각 축 점수가 위 "기대 점수" 범위 안 또는 ±10 이내
- [ ] 9축 가중평균 = `composite` (계산 일치)

### 4.2 Rubric 사용 확인
- [ ] 각 축의 `axis_reports[].markdown` 에 calibration anchor (~15/~35/~55/~75/~90) 와의 비교 명시
- [ ] Reasoning 에 출품작 텍스트의 구체 표현 인용

### 4.3 텍스트 평가 한계 준수
- [ ] 그래픽·오디오 관련 추론으로 점수 올리거나 내리지 않음
- [ ] 샘플 B 의 "16x16 픽셀 아트" 진술이 어떤 축에서도 graphics 점수로 작동하지 않음

### 4.4 종류 분기 확인
- [ ] `template.competitionType === "game"` 인 대회만 게임 preset 사용 가능
- [ ] 일반 대회 (competitionType === "game" + 창업 6축) 도 정상 평가됨 (preset 자유)
