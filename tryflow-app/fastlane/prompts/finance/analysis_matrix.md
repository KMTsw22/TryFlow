## Step 2: 산출물 유형 분석 및 상위 3개 선정

대회들이 실제로 "지원을 받는 형태"를 산출물 유형으로 정규화하고 빈도를 집계했다.

| 산출물 유형 | 등장 대회 | 빈도 | 선정 |
|------------|-----------|:--:|:--:|
| 기획안/제안서 | 금감원, DB IFC(기획안), FSI(공모 트랙) | 3 | ✅ 그룹 A |
| 분석 보고서/연구 논문 | 한국은행, DB IFC(논문), FSI(결과보고서) | 3 | ✅ 그룹 B |
| 발표자료(PT 덱) | 금감원(PPT 1차) + 한국은행·DB IFC·FSI 본선 PT | 4~5 | ✅ 그룹 C |
| 소스코드/모델 산출물 | FSI | 1 | ❌ 빈도 부족 |
| 실적 데이터 | 키움 | 1 | ❌ 빈도 부족 |

→ 상위 3개 산출물(기획안/제안서, 분석 보고서/논문, 발표자료)에 대해 각각 전용 평가 엔진을 설계한다.

## Step 3: 평가 기준 맵 (산출물 × 금융지식 엔진)

설계 원칙: 자격·형식·표절 같은 범용 게이트는 비핵심으로 분리하고, **각 산출물의 "금융 대회로서의 알맹이"를 금융지식 엔진(CORE-FK)을 축으로 평가**한다.

| 기준 ID | 기준명 | 산출물 그룹 | 주력 금융지식 레이어 | 핵심 메커니즘 |
|---------|--------|------------|---------------------|--------------|
| CORE-FK | 금융지식 평가 엔진 | 공유 | L1~L6 전체 | M1~M5 (지식그래프·오개념룰셋·깊이계층·RAG·추론정합성) |
| A1 | 금융 문제 정의의 실재성 | A 기획안/제안서 | L5·L6 | M4 RAG 실재성 검증 |
| A2 | 솔루션의 금융 메커니즘 정합성 | A 기획안/제안서 | L2·L4 | M1 지식그래프 + M4 규제 RAG |
| A3 | 기대효과의 정량적 타당성 | A 기획안/제안서 | L5 | M4 + M5 추정 체인 검증 |
| B1 | 분석 프레임워크 적절성 | B 분석보고서/논문 | L3·L6 | M3 깊이 계층 + 프레임-문제 적합 |
| B2 | 데이터 신뢰성·해석력 | B 분석보고서/논문 | L5 | M4 RAG 출처·수치 대조 |
| B3 | 추론의 논리·인과 엄밀성 | B 분석보고서/논문 | L3·L6 | M5 추론 정합성(인과/상관) |
| C1 | 핵심 메시지의 금융 논리성 | C 발표자료/PT | L1·L2 | M2 오개념 룰셋 + 1장 요약 역검증 |
| C2 | 데이터 시각화 정직성 | C 발표자료/PT | L5 | M4 원본 시계열 대조 + 왜곡 탐지 |
| C3 | 근거-주장 구조 완결성 | C 발표자료/PT | L6 보조 | 슬라이드 역할 분류 + 매칭 |

### 설계 핵심
- **CORE-FK는 산출물당 1회 실행**, A·B·C 3개 기준이 그 결과(layer_scores·violations·claims)를 공유 → LLM 중복 호출 방지.
- 같은 금융 오개념도 산출물마다 다른 메커니즘으로 잡힌다: 기획안=지식그래프, 보고서=추론 정합성, PT=오개념 룰셋.
- **CRITICAL 오개념 1건이면 전 그룹에서 결격(모드 A) 또는 40점 캡(모드 B)** — 금융 대회의 핵심 결격 사유로 격상.

## Step 4: 정교화 v2 반영 사항 (2026-05-13)

| 정교화 축 | 반영 내용 |
|-----------|-----------|
| CORE-FK 엔진 내부 | 오개념 룰셋 50종 완성(8개 카테고리), 지식그래프 노드/엣지 스키마, 깊이 계층 판정 신호, claim 추출 taxonomy(FACT/CAUSAL/NORMATIVE/PREDICTIVE), confidence 산출식 |
| 점수·가중치 체계 | 각 기준에 대회 유형별 가중치·임계값 표, 정규화 공식 `(raw−1)/4×100`, SKIP·결손 시 가중치 재분배, 복수 산출물 합산(핵심70%+보조30%), CRITICAL 40점 캡 |
| LLM 프롬프트 | 9개 기준 전부 few-shot(통과·탈락·경계 3예시)과 엄격한 출력 스키마 명시 |
| 검증·이의제기 | 기준별 REVIEW 트리거 목록, confidence 산출식, 4단계 이의제기 절차(평가 패키지 공개→유형 분류→부분 재평가→이력 보존) |

> 산출물 분량은 v2 정교화로 프롬프트 가이드(1,500~3,500자)를 초과한다. 11개 섹션 + 의사코드 + few-shot + 룰셋 50종을 실제 구현 가능한 수준으로 담은 결과이며, 운영용 요약본이 필요하면 별도 축약 가능.
