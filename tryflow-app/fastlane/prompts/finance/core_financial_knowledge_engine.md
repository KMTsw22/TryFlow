# CORE: 금융지식 평가 엔진 / Financial Knowledge Evaluation Engine

> 모든 산출물별 기준(A·B·C 그룹)이 호출하는 **공유 핵심 엔진**. 지원자가 금융을 실제로 이해하는지를 6개 레이어 × 5개 메커니즘으로 측정한다. 산출물당 1회만 실행되며 결과를 9개 기준이 공유한다.

## 1. 메타데이터
- **엔진 ID**: CORE-FK
- **유형**: 공유 서비스 (산출물별 기준이 API 형태로 호출)
- **자동화 수준**: 반자동 — 객관 메커니즘(M1·M2·M4) 자동, 정성 메커니즘(M3·M5) LLM, CRITICAL 건은 사람 검토
- **출처 근거**: 5개 벤치마크 대회 중 4개(한국은행·금감원·DB IFC·FSI)가 금융 이해도를 핵심 심사 항목으로 사용

## 2. 금융지식 6개 평가 레이어
| 레이어 | 평가 대상 | 대표 실패 사례 | 담당 메커니즘 |
|--------|-----------|----------------|--------------|
| L1 용어 정확성 | 금융 용어를 정확한 의미로 사용 | "유동성"과 "수익성" 혼용 | 용어집 매칭 |
| L2 개념 정합성 | 개념 간 관계가 올바름 | "금리↑ → 채권가격↑" | M1, M2 |
| L3 메커니즘 이해 | 작동 원리·인과 체인을 설명 | "금리 올리면 물가 잡힌다"만 진술 | M3, M5 |
| L4 규제·제도 이해 | 제도적 제약·인가 체계를 인지 | 무인가 금융업 영위 전제 | M4 |
| L5 데이터 해석력 | 시장·통계 데이터를 정확히 해석 | CPI 상승률과 물가 수준 혼동 | M2, M4 |
| L6 맥락 적용력 | 지식을 해당 문제에 적용 | 교과서 정의만 나열 | M3, M5 |

## 3. 금융적 주장 추출 (Claim Extraction)
엔진은 평가 전, 제출물에서 "금융적 주장"만 분리한다. 주장 유형별로 적용 메커니즘이 달라진다.

| 주장 유형 | 정의 | 적용 메커니즘 |
|-----------|------|--------------|
| 사실 주장 (FACT) | 검증 가능한 수치·제도·정의 | M2(용어), M4(RAG 대조) |
| 인과 주장 (CAUSAL) | "A가 B를 유발/억제한다" | M1(지식그래프 부호 대조), M5(전이경로 검사) |
| 규범 주장 (NORMATIVE) | "~해야 한다" 정책 권고 | M5(추론 도출 검사) — 사실/인과로 환원해 검증 |
| 예측 주장 (PREDICTIVE) | "향후 ~할 것이다" | M5(가정 명시 여부), M4(기준 데이터) |

추출 결과: `claim = {text, type, financial_entities[], asserted_relation, slide/page_ref}`

## 4. 평가 메커니즘 5종

### M1 — 금융 지식그래프 대조
인과·관계 주장을 구조화된 지식그래프와 대조해 부호·방향 충돌을 탐지한다.

**노드 타입**: `CONCEPT`(기준금리, 통화량), `INDICATOR`(CPI, 환율, GDP), `INSTRUMENT`(채권, 주식, 적금, 보험), `REGULATION`(금소법, 자본시장법), `ENTITY`(한국은행, 시중은행)

**엣지 타입**:
| 엣지 | 의미 | 속성 |
|------|------|------|
| `CAUSES(+)` / `CAUSES(−)` | 인과, 부호 포함 | lag(시차), confidence, transmission_path |
| `CORRELATES` | 비인과 상관 | — (인과 주장이 이 엣지에만 의존하면 비약) |
| `COMPOSED_OF` | 구성 관계 | — |
| `TRADES_OFF` | 상충 관계 | (위험↔수익, 유동성↔수익성) |
| `REGULATED_BY` | 규제 적용 | regulation_id |

**예시 서브그래프 (통화정책)**:
```
기준금리 --CAUSES(−), lag=0--> 통화량
기준금리 --CAUSES(+), lag=1~3M--> 시장금리 --CAUSES(−)--> 채권가격
기준금리 --CAUSES(−), lag=6~18M, path=[대출금리↑→소비·투자↓→수요↓]--> 물가
위험 --CAUSES(+)--> 기대수익        (위험프리미엄)
환율(원/달러) --CAUSES(−)--> 원화가치
```
**판정**: 주장의 (from, to, 부호)가 그래프 엣지와 반대면 `GRAPH_CONTRADICTION` — 부호 충돌은 CRITICAL, 시차 무시는 MAJOR.

### M2 — 오개념 룰셋 (50종)
`(트리거 패턴, 심각도, 정정)` 규칙으로 결정론적 탐지. 학계 합의가 확립된 항목만 포함하며 분기별 갱신. 전체 50종은 §8.

### M3 — 깊이 계층 채점
각 금융 논증을 Tier 0~3으로 분류. 판정은 LLM이 하되, 아래 **관찰 가능한 신호**로 가이드한다.

| Tier | 점수 | 정의 | 판정 신호 |
|------|:--:|------|-----------|
| Tier 0 | 0 | 용어 나열 (recitation) | 정의문만 존재, 문제와 연결 없음 |
| Tier 1 | 2 | 개념을 맥락에 맞게 사용 | 개념을 이 문제에 적용, 단 단일 문장 인과 |
| Tier 2 | 4 | 메커니즘·인과 체인 설명 | 전이경로(A→B→C) 명시, 매개변수 등장 |
| Tier 3 | 5 | 시차·전이·2차효과·트레이드오프 고려 | "단기엔 X, 시차 후 Y", 반대 효과·부작용 언급 |

### M4 — RAG 사실 검증
수치·규제·시장 데이터를 권위 출처에서 retrieve해 대조한다.

| 검증 대상 | 1차 출처 | 허용 오차 |
|-----------|----------|-----------|
| 거시지표(금리·물가·환율·GDP) | 한국은행 ECOS | 확정치 ±0%, 잠정치 ±0.2%p |
| 통계(인구·가계·고용) | 통계청 KOSIS | ±1% |
| 규제·법령 | 국가법령정보센터, 금융위·금감원 감독규정 | 시행 여부·조항 일치 |
| 금융 민원·시장 통계 | 금감원·예보·각 협회 공시 | ±5% |
판정: `MATCH` / `STALE`(시점 경과) / `MISMATCH`(값 불일치) / `UNVERIFIABLE`(출처 미상)

### M5 — 추론 정합성 검사
사실이 맞아도 추론이 틀릴 수 있다. LLM에 아래 항목을 명시 질의한다.
1. **인과 vs 상관**: 인과 주장의 근거가 `CORRELATES` 엣지뿐인가
2. **전이경로 부재**: 인과 주장에 매개 경로(transmission path)가 제시되었는가
3. **범위 비약**: 미시 결론을 거시로, 부분균형을 일반균형으로 확대했는가
4. **교란변수 무시**: 제3변수 가능성을 배제했는가
5. **결론 도출**: 결론이 전제에서 실제로 따라 나오는가

## 5. 엔진 처리 파이프라인
```
function evaluate_financial_knowledge(submission, artifact_type):
    step 1: claims = extract_financial_claims(submission)          # §3
    step 2: L1 = term_accuracy(claims, finance_glossary)
    step 3: g = M1_graph_check(claims.CAUSAL)
            mc = M2_misconception_ruleset(claims)
            L2 = combine(g, mc)                                   # 0~5
    step 4: critical = count(violations where severity == CRITICAL)
    step 5: L3 = M3_depth_tier(claims.CAUSAL + claims.NORMATIVE)
    step 6: L4 = M4_rag_verify(claims where type==FACT and domain in [규제,제도])
            L5 = M4_rag_verify(claims where type==FACT and domain==데이터)
    step 7: L6 = M3_context_application(claims, submission.problem)
            reasoning = M5_reasoning_coherence(submission.argument_chains)
    step 8: w = artifact_layer_weights[artifact_type]             # §6
            raw = Σ(L_i × w_i)
            raw = raw × reasoning_factor   # reasoning 0.7~1.0 배율
    step 9: if critical >= 1: raw = min(raw, 2.0)                 # CRITICAL 게이트
    step 10: conf = confidence_formula(...)                       # §7
             if conf < 0.6 or |objective_score − llm_score| >= 2: review = true
    return { score: round1(raw), layer_scores, violations, tier_map, reasoning, confidence: conf, review }
```

## 6. 산출물별 레이어 가중치
| 레이어 | 기획안/제안서(A) | 분석보고서/논문(B) | PT덱(C) | 근거 |
|--------|:--:|:--:|:--:|------|
| L1 용어 정확성 | 10% | 10% | 20% | PT는 압축돼 용어 오용이 빈번 |
| L2 개념 정합성 | 25% | 15% | 30% | 제안·발표는 개념 정합이 핵심 |
| L3 메커니즘 이해 | 20% | 25% | 15% | 보고서는 메커니즘 설명이 본령 |
| L4 규제·제도 이해 | 25% | 15% | 10% | 제안은 규제 적합성이 실현 관건 |
| L5 데이터 해석력 | 5% | 20% | 20% | 보고서·PT는 데이터 의존도 높음 |
| L6 맥락 적용력 | 15% | 15% | 5% | PT는 맥락 적용을 깊게 못 담음 |

## 7. Confidence 산출식
```
confidence = 0.35 × signal_agreement      # M1·M2·M4(객관)와 M3·M5(LLM) 점수 일치도
           + 0.25 × coverage              # 추출된 claim 중 검증 가능했던 비율
           + 0.20 × rag_hit_rate          # M4가 출처 대조에 성공한 비율
           + 0.20 × llm_self_consistency  # 동일 프롬프트 3회 샘플링 점수 분산의 역
```
- `confidence < 0.6` → REVIEW. `0.6 ≤ confidence < 0.75` → 점수는 채택하되 HOLD 후보 표시.

## 8. 오개념 룰셋 (전체 50종)
심각도: CRITICAL = 금융 논리의 근본 위반 / MAJOR = 분석 결론을 왜곡 / MINOR = 부정확하나 결론 영향 제한적

### A. 통화·금리 (통화정책)
| ID | 트리거(요지) | 심각도 | 정정 |
|----|-------------|--------|------|
| MC-01 | 기준금리 인상 → 통화량 증가 | CRITICAL | 기준금리 인상은 통화량 축소 압력 |
| MC-02 | 기준금리와 시장금리를 동일시 | MAJOR | 기준금리는 정책금리, 시장금리는 시차·가산금리로 별개 |
| MC-03 | 통화량 증가 → 즉시 물가 상승 (시차 무시) | MAJOR | 통화정책은 6~18개월 시차로 물가에 전이 |
| MC-04 | 한국은행이 시중 대출금리를 직접 결정 | MAJOR | 한은은 기준금리만 결정, 대출금리는 시장이 결정 |
| MC-05 | 양적완화 = 현금(화폐) 발행 | MAJOR | QE는 자산매입을 통한 지급준비금 확대 |
| MC-06 | 콜금리와 기준금리를 동일시 | MINOR | 콜금리는 단기 시장금리, 기준금리 타깃과 구분 |
| MC-07 | 금리 인하는 항상 경기 부양 (유동성 함정 무시) | MINOR | 유동성 함정에서는 효과 제한 |

### B. 채권·주식
| ID | 트리거 | 심각도 | 정정 |
|----|--------|--------|------|
| MC-08 | 금리 상승 → 채권가격 상승 | CRITICAL | 금리와 채권가격은 역관계 |
| MC-09 | 채권 수익률(YTM)과 표면금리(쿠폰)를 동일시 | MAJOR | YTM은 시장가격 반영, 쿠폰은 고정 |
| MC-10 | 듀레이션이 길수록 금리위험이 작다 | MAJOR | 듀레이션이 길수록 금리위험이 큼 |
| MC-11 | 액면가 매입 채권은 항상 원금 보장 | MAJOR | 발행사 부도위험(신용위험) 존재 |
| MC-12 | PER이 낮으면 무조건 저평가 | MINOR | 업종·성장성·위험에 따라 해석 달라짐 |
| MC-13 | 배당수익률이 높을수록 좋은 주식 | MINOR | 주가 하락으로 인한 착시 가능 |
| MC-14 | 주가 상승 = 기업 가치 증가 (항상) | MINOR | 수급·심리 요인 존재 |

### C. 환율·국제금융
| ID | 트리거 | 심각도 | 정정 |
|----|--------|--------|------|
| MC-15 | 환율(원/달러) 상승 = 원화 강세 | CRITICAL | 원/달러 상승 = 원화 약세 |
| MC-16 | 환율 상승은 항상 수출에 유리 | MINOR | J커브 효과·수입물가 상승 고려 |
| MC-17 | 경상수지 흑자 → 환율(원/달러) 상승 | MAJOR | 흑자는 원화 강세(환율 하락) 압력 |
| MC-18 | 금리 인상 → 자본 유출 | MAJOR | 금리 인상은 자본 유입 압력 |
| MC-19 | 외환보유액은 많을수록 항상 좋다 | MINOR | 보유 비용·기회비용 존재 |

### D. 위험·수익·포트폴리오
| ID | 트리거 | 심각도 | 정정 |
|----|--------|--------|------|
| MC-20 | 무위험 고수익 보장 | CRITICAL | 위험-수익 상충 원칙 위반 |
| MC-21 | 분산투자 = 헤지 | MAJOR | 분산은 비체계적 위험 축소, 헤지는 특정 위험 상쇄 |
| MC-22 | 분산투자로 모든 위험 제거 가능 | MAJOR | 체계적(시장) 위험은 잔존 |
| MC-23 | 과거 수익률이 미래 수익률을 보장 | MAJOR | 과거 성과는 미래를 보장하지 않음 |
| MC-24 | 변동성이 낮으면 위험이 없다 | MAJOR | 꼬리위험·유동성위험 존재 |
| MC-25 | 레버리지는 수익만 키운다 | MAJOR | 손실도 동일 배율로 확대 |
| MC-26 | 베타가 높으면 항상 수익이 높다 | MAJOR | 베타는 기대수익이 아닌 시장민감도 |
| MC-27 | 표준편차가 같으면 위험이 같다 | MINOR | 분포 형태·하방위험 차이 |

### E. 명목·실질·인플레이션
| ID | 트리거 | 심각도 | 정정 |
|----|--------|--------|------|
| MC-28 | CPI 상승률 = 물가 수준 | CRITICAL | 상승률(변화)과 수준(절대값)은 다른 개념 |
| MC-29 | 명목수익률을 실질수익률처럼 사용 | MAJOR | 실질 = 명목 − 물가상승률 |
| MC-30 | 명목 GDP 증가 = 실질 성장 | MAJOR | 물가 효과 제거해야 실질 성장 |
| MC-31 | 임금상승률 > 물가상승률이면 실질임금 하락 | MAJOR | 그 경우 실질임금은 상승 |
| MC-32 | 디플레이션은 물가 하락이니 항상 좋다 | MAJOR | 부채 디플레이션·소비 지연 등 폐해 |
| MC-33 | 인플레이션 = 통화량 증가 (동의어) | MINOR | 통화량은 한 요인일 뿐 |

### F. 규제·제도·인가
| ID | 트리거 | 심각도 | 정정 |
|----|--------|--------|------|
| MC-34 | 인가 없이 금융업 영위 가능 전제 | CRITICAL | 금융업은 업권별 인가·등록 필수 |
| MC-35 | 모든 금융상품이 예금자보호 대상 | MAJOR | 펀드·주식·일부 상품은 비대상 |
| MC-36 | 예금자보호 한도가 무제한 | MAJOR | 금융회사별 한도 존재 |
| MC-37 | 금소법상 6대 판매규제를 무시 | MAJOR | 적합성·적정성·설명의무 등 준수 필요 |

## 10. 지식 베이스 요구사항
- **금융 용어집**: 한국은행 경제금융용어 700선 + 감독용어, 2026년 기준 최신화, 동의어·약어 매핑 포함
- **지식그래프**: 핵심 개념 약 300노드 + 관계 엣지 약 800개, 분기(통화/은행/증권/보험/국제), 엣지마다 출처·confidence 기재
- **오개념 룰셋**: 50종(§8), 분기별 갱신, 신규 룰은 학계·감독당국 자료로 검증 후 추가
- **RAG 소스**: 한국은행 ECOS, 통계청 KOSIS, 국가법령정보센터, 금융위·금감원 감독규정, 예보·각 협회 공시

## 11. 한계 및 운영 원칙
- 일반 LLM은 최신 규제·시장 상황을 모를 수 있다 → L4·L5는 RAG 결과를 우선하고 LLM 단독 판정 금지
- 도메인 깊이(L3·L6)는 전문가 간 견해차가 있다 → CRITICAL·상위권 건은 도메인 전문가 검토 필수
- "오개념"(명백한 오류)과 "견해차"를 엄격히 구분 — 룰셋은 학계 합의가 확립된 항목만 포함
- 지식그래프·룰셋이 미수록한 신종 금융구조는 `UNVERIFIABLE`로 두고 LLM 판정과 교차 검증, 자동 결격 금지
- 모든 판정에 quote(원문)·correction(정정)·ref(위치)를 첨부하여 이의제기에 대응
